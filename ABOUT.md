# About This Boilerplate

This repository is a reusable web + CMS boilerplate. It combines a Next.js frontend and a Sanity Studio CMS in one npm-workspace monorepo.

It is intentionally generic. Do not treat the visible homepage, sample blog posts, or sample FAQ items as permanent product content. They exist so developers can test the structure before adapting it for a real project.

## High-Level Architecture

```text
Browser
  -> Next.js app in apps/web
      -> Sanity queries through apps/web/src/sanity
      -> SEO metadata through apps/web/src/seo
      -> JSON-LD through apps/web/src/schema
      -> routes under apps/web/src/app/[locale]

Editors
  -> Sanity Studio in apps/studio
      -> schemaTypes/documents define document models
      -> schemaTypes/objects define reusable field groups
      -> structure/index.ts defines the editor sidebar

Sanity publish
  -> POST /api/revalidate
      -> validates SANITY_REVALIDATE_SECRET
      -> revalidates tagged Next.js data
```

## Root Files And Folders

```text
monorepo/
|-- AGENTS.md
|-- ABOUT.md
|-- SETUP.md
|-- README.md
|-- package.json
|-- package-lock.json
|-- .gitignore
|-- .claude/
|-- node_modules/
`-- apps/
    |-- web/
    `-- studio/
```

### `AGENTS.md`

Root-level instructions for agents and developers working on the boilerplate. It explains the project boundaries, core schema types, and the rule that the base structure should not be changed unless explicitly requested.

### `ABOUT.md`

This file. It is the long-form structure guide for new developers.

### `SETUP.md`

Detailed setup guide covering cloning, zip setup, env files, Sanity content cleanup, local development, Vercel deployment, Studio deployment, CORS, preview, and revalidation.

### `README.md`

Shorter general handoff guide. Use it for a quick overview. Use `ABOUT.md` and `SETUP.md` for complete detail.

### `package.json`

Root npm workspace configuration. It defines `apps/web` and `apps/studio` as workspaces and exposes shared commands:

- `npm run dev`
- `npm run dev:web`
- `npm run dev:studio`
- `npm run build`
- `npm run build:web`
- `npm run build:studio`

### `package-lock.json`

npm lockfile for the full workspace. Do not replace it with yarn, pnpm, or bun lockfiles.

### `.gitignore`

Keeps build output, local env files, local editor state, generated artifacts, and temporary files out of git.

### `.claude/`

Local agent/editor state. This is ignored and not part of the reusable boilerplate.

### `node_modules/`

Installed dependencies. Generated by `npm install`. Not committed.

## `apps/web/src/app`

Next.js App Router routes, layouts, global CSS, and API routes.

```text
apps/web/src/app/
|-- globals.css
|-- robots.js
|-- sitemap.js
|-- api/
|   |-- draft-mode/
|   |   |-- enable/route.js
|   |   `-- disable/route.js
|   `-- revalidate/route.js
`-- [locale]/
    |-- layout.js
    |-- page.js
    |-- faq/page.js
    |-- blog/
    |   |-- page.js
    |   |-- [slug]/page.js
    |   `-- category/[slug]/page.js
    `-- author/[slug]/page.js
```

### `globals.css`

Global CSS entrypoint. Includes Tailwind v4 import, theme variables, skip-link behavior, and imported shared styles.

### `robots.js`

Builds `/robots.txt`.

### `sitemap.js`

Builds `/sitemap.xml` from core routes and Sanity content.

### `api/draft-mode/enable/route.js`

Enables Next.js Draft Mode. Used by Sanity Presentation preview and direct preview links. Requires `SANITY_API_READ_TOKEN` and `SANITY_PREVIEW_SECRET`.

### `api/draft-mode/disable/route.js`

Disables Draft Mode and redirects back to the public site.

### `api/revalidate/route.js`

Receives Sanity webhooks and revalidates tagged data. Requires `SANITY_REVALIDATE_SECRET`.

### `[locale]/layout.js`

Root locale layout. It validates the locale, loads Site Settings, renders navbar/footer, injects site-wide JSON-LD, mounts analytics, and enables Visual Editing in Draft Mode.

### `[locale]/page.js`

Generic homepage placeholder. Page content is code-owned; SEO, FAQ, and schema settings are Sanity-owned through `landingPageSeo` slug `/`.

### `[locale]/faq/page.js`

FAQ page. It reads FAQ page SEO and selected FAQ items from Sanity.

### `[locale]/blog/page.js`

Blog listing page with category filtering.

### `[locale]/blog/[slug]/page.js`

Blog post detail page. Handles old slug redirects, metadata, JSON-LD, author data, FAQ blocks, and related posts.

### `[locale]/blog/category/[slug]/page.js`

Category archive page for blog posts.

### `[locale]/author/[slug]/page.js`

Author profile page and posts by author.

### `schemaTypes/index.ts`

Central schema registry. Merges documents and objects into one array for `sanity.config.ts`.

Wraps Sanity's publish action for redirect-aware documents. It records old blog slugs so frontend routes can redirect old URLs.

Custom Studio UI:

- `BlogPostsPane.tsx`: custom blog post list pane.

- `PreviewPane.tsx`: editor preview pane.

- `SeoChecklist.tsx`: SEO checklist panel.

### `documents/*`

Core CMS documents. Do not rename or remove these without an explicit migration plan.

- `author.ts`: author profile and E-E-A-T data.
- `blogPost.ts`: blog post content, SEO, schema, status, and slug history.
- `category.ts`: blog taxonomy.
- `faqItem.ts`: reusable FAQ item.
- `index.ts`: document exports.
- `landingPageSeo.ts`: SEO/schema/FAQ config for code-built pages.

Reusable field groups:

- `aiSeo.ts`: quick answer and key takeaways fields.

- `faqList.ts`: embedded FAQ list object.
- `imageWithMeta.ts`: image with alt/caption/meta fields.
- `index.ts`: object exports.
- `link.ts`: internal/external link object.
- `portableText.ts`: rich text schema and custom blocks.
- `redirectSettings.ts`: redirect-related fields.
- `schemaConfig.ts`: primary schema type selection.
- `searchIndexSettings.ts`: reserved search settings.
- `seo.ts`: metadata and social sharing fields.
- `socialDefaults.ts`: social profile defaults.

## Core Structure That Should Stay Stable

For ordinary project customization, preserve these contracts:

- Route base: `apps/web/src/app/[locale]`.
- Blog routes: `/blog`, `/blog/[slug]`, `/blog/category/[slug]`.
- Author route: `/author/[slug]`.
- FAQ route: `/faq`.
- Draft Mode endpoints: `/api/draft-mode/enable` and `/api/draft-mode/disable`.
- Revalidation endpoint: `/api/revalidate`.
- Sanity query layer in `apps/web/src/sanity`.
- Sanity schema layer in `apps/studio/schemaTypes`.
- Core Sanity documents listed above.

Add to the structure when needed. Do not collapse web and Studio into one app, move routes out of `[locale]`, or bypass the Sanity query layer unless explicitly asked.
