# Next.js + Sanity Studio Monorepo — Complete Setup Specification

> **For AI agents:** This document is a complete, self-contained specification. Follow every section in order. All file contents are given verbatim — do not infer, invent, or add anything not listed here. Use only npm (no yarn, pnpm, or bun). All paths are relative to the monorepo root unless stated otherwise.

---

## Overview

This monorepo contains two apps managed via npm workspaces:

| App | Path | Tech | Port |
|---|---|---|---|
| Frontend | `apps/web` | Next.js 16, React 19, Tailwind v4 | 3000 |
| CMS | `apps/studio` | Sanity Studio v5 | 3333 |

**Key constraints:**
- npm only — no other package managers
- JavaScript (not TypeScript) in `apps/web`
- TypeScript in `apps/studio` (Sanity default)
- Minimal — no boilerplate, no placeholder content, no unused packages
- Single root-level `.gitignore` and `README.md`; none inside sub-apps
- `npm run dev` runs both apps in parallel via `concurrently`

## Part 5 — Installation

Run from the monorepo root:

```bash
npm install
```

This single command:
- Installs all dependencies for both workspaces
- Hoists shared packages (`react`, `react-dom`, `eslint`) to root `node_modules/`
- Installs workspace-specific packages where needed
- Installs `concurrently` at root

> **Do not run `npm install` inside `apps/web` or `apps/studio`** — always run from root.

## Part 6 — Gotchas and rules

### npm workspace script syntax
```bash
# Correct — workspace flag
npm run dev -w apps/web

# Wrong — this runs sequentially and blocks since next dev never exits
npm run dev --workspaces
```

### `.env.example` must NOT be in `.gitignore`
The pattern `.env*.local` covers `.env.local`, `.env.production.local`, etc., but NOT `.env.example`. This is intentional — the example file is a safe committed template.

### No nested `.git` directories
`create-next-app` and `sanity init` both run `git init` inside their output directory. After scaffolding, remove any nested `.git` folders so only the root repo exists:

```bash
# macOS / Linux
rm -rf apps/web/.git
rm -rf apps/studio/.git

# Windows (PowerShell)
Remove-Item -Recurse -Force apps\web\.git
Remove-Item -Recurse -Force apps\studio\.git
```

### Tailwind v4 import syntax
Tailwind v4 changed the CSS API. Use:
```css
@import "tailwindcss";
```
Not:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### `next-sanity` v13+ API
`next-sanity` v13 introduced the Live Content API. The correct imports are:
```js
import { createClient } from 'next-sanity'        // client
import { defineLive } from 'next-sanity'           // live fetching
```
Not the older `@sanity/client` package — `next-sanity` re-exports everything needed.

### Studio nested directory bug
`sanity init --output-path studio` sometimes creates `apps/studio/<project-name>/` instead of putting files directly in `apps/studio/`. If this happens, move all files up one level and delete the empty subfolder before running `npm install`.

## Part 8 — How to extend

### Add a Sanity schema type

1. Create `apps/studio/schemaTypes/post.ts` (example):

```ts
import { defineType, defineField } from 'sanity'

export default defineType({
  name: 'post',
  title: 'Post',
  type: 'document',
  fields: [
    defineField({ name: 'title', type: 'string' }),
    defineField({ name: 'body', type: 'text' }),
  ],
})
```

2. Register it in `apps/studio/schemaTypes/index.ts`:

```ts
import post from './post'
export const schemaTypes = [post]
```

### Fetch Sanity content in Next.js

```js
// In a Server Component
import { sanityFetch } from '@/sanity/lib/live'

const QUERY = `*[_type == "post"]{ title, body }`

export default async function Page() {
  const { data: posts } = await sanityFetch({ query: QUERY })
  return <ul>{posts.map(p => <li key={p.title}>{p.title}</li>)}</ul>
}
```

### Enable real-time updates

Add `<SanityLive />` once to the root layout:

```js
// apps/web/src/app/layout.js
import "./globals.css";
import { SanityLive } from "@/sanity/lib/live";

export const metadata = { title: "My App", description: "My Next.js application" };

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}
        <SanityLive />
      </body>
    </html>
  );
}
```

### Deploy Sanity Studio

```bash
npm run build:studio
npm run deploy -w apps/studio
```

This hosts Studio on `https://<your-project>.sanity.studio`.
